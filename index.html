Weather Events in the United States with the Greatest Human Population Health and Economic Impacts (1996-2011)

reproducible research | weather forecasting | storm damage | extreme weather
Author

Published

November 19, 2023

Synopsis

Severe weather events, such as storms and floods, can result in fatalities, injuries, and property damage. The purpose of the research reported here is to identify the weather events with the greatest public health and economic impacts. The data used for the report are from the U.S. National Oceanic and Atmospheric Administration’s (NOAA) storm database. The storm database tracks characteristics of major storms and weather events in the United States, including when and where they occur, as well as estimates of any fatalities, injuries, and property damage. The events in the database start in 1950 and end in November 2011. The analysis reported here finds that floods cause the most deaths and tsunamis the most injuries. In second place are storm surges/tides for death and floods for injuries. Storms (surges/tides and hurricanes) cause the most damage to property and crops with floods in second place. The damage to property is 10x as great as that to crops. Overall, winter weather has less of an impact than warmer weather storms.

Reduce Data

In the earlier years of the database there are generally fewer events recorded, most likely due to a lack of good records. More recent years tend to be more complete. In fact, several early decades only include tornado data, which would skew the results of any analysis falsely towards tornadoes having the most impact. The following code chunk removes rows prior to 1996.

Code 
#reduce data rows to include only complete data, post 1995
storm.subDate <- storm.data
storm.subDate$BGN_DATE <- mdy_hms(storm.subDate$BGN_DATE)
storm.subDate <- subset(storm.subDate, BGN_DATE > "1995-12-31")
#Reduce data columns
storm.subCol <- storm.subDate %>% select(c
    (EVTYPE,FATALITIES,INJURIES,PROPDMG,PROPDMGEXP,CROPDMG,CROPDMGEXP))

Wrangle Damage Data

The following code chunk wrangles the data for property damage. Damage to property and crops are each reported in paired columns–a numeric value and its exponent in string format. The first bit of code chunk checks the values found the exponent columns. All are thousands, millions, and billions, except one error (an invalid entry of 0) which is corrected. Because the variables are reported in both lower and upper case, all variables of interest were converted to lower case–the other columns of interst and column names were also converted for downstream easy of data handling. The final bit of the code chunk replaces the exponent by a value (10^6 for millions), and creates a new variable with the value resulting from multiplying the numeric column by the transformed exponent. The new columns (prop.c and prop.p) reflect the actual amount of damage in dollars. Finally, a new smaller dataset is created with only the five variables/columns of interest.

Wrangle Weather Event Type Data

This code chunk examines the event type column. Out of a possible 48 Storm Data Event types in Table 2.1.1 of the storm data documentation (https://d396qusza40orc.cloudfront.net/repdata%2Fpeer2_doc%2Fpd01016005curr.pdf), there were over 400 unique values in the column evtype. An examination of the data indicated that data entry errors were the primary problem. Values were replaced using the official storm data event names and, when indeterminate, were listed as other. The final number of events was 42.

The following code chunk aggregates sums of injuries and fatalities by weather event and then produces a table of the top five.

Code 
storm.fat <- aggregate(fatalities~ evtype, storm, sum)
storm.inj <- aggregate(injuries~ evtype, storm, sum)
fatalities <- arrange(storm.fat, desc(fatalities),evtype)[1:5,]
injuries <- arrange(storm.inj, desc(injuries),evtype)[1:5,]
fatalities$evtype <- factor(fatalities$evtype, levels = fatalities$evtype)
injuries$evtype <- factor(injuries$evtype, levels = injuries$evtype)

The following code chunk creates a panel plot with one bar plot for total fatalities due to weather events and total injuries due to weather events. Flooding is the biggest cause of fatalities in the US and tsunamis (storms and winds) are the biggest cause of injuries in the US. Three of the top five weather events causing fatalities are storm-related and the other two are temperature related. While both cold and heat cause fatalities, only heat made the top five for injuries. Plots are made using the color-blind friendly palette, viridis.

Code 
p1 <- ggplot(fatalities, aes(x=reorder(evtype, fatalities),
    y=fatalities,
    fill=fatalities))+
    scale_fill_viridis_c()+
    geom_bar(stat="identity")+
    theme_classic2()+
    theme(axis.text.x = element_text(angle = 45, hjust = 1)) + ylab("Fatalities") + xlab("Weather Events")+
    theme(legend.position = "none")+
    coord_flip()
p2 <- ggplot(injuries, aes(x=reorder(evtype, injuries),
    y=injuries, fill=injuries))+
    scale_fill_viridis_c()+
    geom_bar(stat="identity")+
    theme_classic2()+
    theme(axis.text.x = element_text(angle = 45, hjust = 1)) + ylab("Injuries") + xlab("")+
    theme(legend.position = "none")+
    coord_flip()
grid.arrange(p1,p2, nrow=1,
    top = textGrob("Weather Events (1996-2011) with the Greatest Public Health Impact"))

The following code chunk creates a panel plot with one bar plot for total property damage due to weather events and total crop damage due to weather events. Total damage is reduced to millions for easy of display in the plot. Storms are the biggest cause of both types of damage–with storm surges and tides created mroe property damage and hurricanes creating more crop damage. Floods re the number two position for both. Property damage (e.g., house, boats) are 10x more costly than crop damages (see x-axis values). Plots are made using the color-blind friendly palette, viridis.
